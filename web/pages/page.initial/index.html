1 - Para que serve o commit e qual a melhor forma de utiliza-lo?

R: O git commit serve para você guardar as informações desenvolvidas por um programador, isso será feito a cada alteração feita e que estiver funcionando.
Que ajudam na manutenção do sistema, porque os logs de cada alteração irá ficar salvo.

2 - Qual a diferença entre commit e push ?

R: O git commit tem como principal papel comunicar as alterações que estamos fazendo no código. Sempre inserimos uma mensagem no commit para saber
o que faz o código que acabamos de inserir. Já o git pull quando finalizamos toda a tarefa que estamos realizando precisamos enviar o nosso código para
o repositório remoto então o commit e o push eles se complementam um manda as alterações e o outro manda para o repositório as alterações.

3 - O que é tag ?

R: Tag em inglês quer dizer etiqueta, quer dizer que utiliza das etiquetas para marcar cada alteração do sistema ajudam a organizar os commits feitos
pelo desenvolvedor. E saber qual versão está o sistema e seus anteriores, na hora que precisar achar as alterações feitas em tal dia ir direto na versão
que queira.

4 - O que é branch ?

R: Branch é uma ramificação para as alterações feitas no arquivos do projeto. Ele é bem util para quando você deseja adicionar um recurso (feature) ou corrigir
erros. Gerando assim uma nova ramificação garantindo que seu codigo que está funcionando não seja mesclado com aquele que está sendo corrigido. No fim com tudo
devidamente testado você pode pegar e mesclar a branch feature ou a sua correção na ramificação principal do projeto.

5 - Como resolver possiveis conflitos no git ?

R: Bom primeiramente você precisa saber que tipo de erro ocorreu, pra dai sim achar uma solução coerente. Temos erros de quando dois desenvolvedores estão fazendo
algo na mesma branch, isso futuramente pode dar conflito nos codigos feitos em seus respectivos arquivos do projeto. Temos tambem varios comandos que podem ajudar a você
corrigir conflitos
